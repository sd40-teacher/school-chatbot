═══════════════════════════════════════════════
  📦 GitHub 업로드 가이드 (단계별 상세 설명)
═══════════════════════════════════════════════

이 파일은 선생님께서 GitHub에 파일을 업로드하는 방법을 
단계별로 상세하게 안내합니다.

═══════════════════════════════════════════════
  ✅ STEP 1: 파일 준비 (2분)
═══════════════════════════════════════════════

1. 다운로드한 파일 확인:
   
   school-chatbot/
   ├── data/
   │   └── school_docs/
   │       └── README.md
   ├── .streamlit/
   │   └── secrets.toml
   ├── app.py
   ├── rag_engine.py
   ├── requirements.txt
   ├── README.md
   └── 업로드_가이드.txt (이 파일)

2. PDF 파일 추가:
   
   data/school_docs/ 폴더에
   [학교소개.pdf] 파일을 복사하세요!
   
   결과:
   data/school_docs/
   ├── README.md
   └── 학교소개.pdf  ← 추가됨!

═══════════════════════════════════════════════
  ✅ STEP 2: GitHub 저장소 생성 (3분)
═══════════════════════════════════════════════

1. GitHub 접속
   https://github.com/

2. 로그인
   (이미 계정 있으시죠?)

3. 새 저장소 만들기
   - 우측 상단 "+" 버튼 클릭
   - "New repository" 선택

4. 저장소 설정
   Repository name: school-chatbot
   Description: 성동글로벌경영고 AI 챗봇
   ✅ Public 선택
   ❌ Add a README file 체크 해제!
   
   → "Create repository" 클릭

5. 생성 완료!
   빈 저장소가 만들어집니다.

═══════════════════════════════════════════════
  ✅ STEP 3: 파일 업로드 (5분)
═══════════════════════════════════════════════

방법 1: 웹 인터페이스 사용 (추천!)
────────────────────────────────────

1. 저장소 페이지에서
   "uploading an existing file" 링크 클릭
   (중간쯤에 보임)

2. 파일 선택
   - school-chatbot 폴더 전체를 선택
   - 드래그 앤 드롭
   
   또는
   
   - "choose your files" 클릭
   - 모든 파일/폴더 선택

3. Commit 메시지 입력 (선택)
   "초기 업로드" 또는 그냥 기본값 사용

4. "Commit changes" 클릭!

5. 업로드 완료 대기 (1-2분)


방법 2: 폴더별 업로드 (대안)
────────────────────────────────────

만약 전체 업로드가 안 되면:

1. requirements.txt 먼저 업로드
   - "Add file" → "Create new file"
   - 파일명: requirements.txt
   - 내용 복사 붙여넣기
   - Commit

2. app.py 업로드
   - "Add file" → "Create new file"
   - 파일명: app.py
   - 내용 복사 붙여넣기
   - Commit

3. rag_engine.py 업로드
   (위와 동일)

4. .streamlit/secrets.toml 업로드
   - 파일명: .streamlit/secrets.toml
   - 내용 붙여넣기
   - Commit

5. data/school_docs/ 폴더 만들기
   - "Add file" → "Upload files"
   - PDF 파일 드래그
   - Commit

═══════════════════════════════════════════════
  ✅ STEP 4: PDF 파일 확인 (1분)
═══════════════════════════════════════════════

1. data/school_docs/ 폴더로 이동

2. 다음 파일들이 있는지 확인:
   ✅ README.md
   ✅ 학교소개.pdf

3. 없으면:
   - "Upload files" 클릭
   - 학교소개.pdf 업로드
   - Commit changes

═══════════════════════════════════════════════
  ✅ STEP 5: Streamlit Cloud 배포 (5분)
═══════════════════════════════════════════════

1. Streamlit Cloud 접속
   https://streamlit.io/cloud

2. 로그인
   "Continue with GitHub" 클릭

3. 새 앱 만들기
   "New app" 버튼 클릭

4. 설정
   Repository: school-chatbot
   Branch: main
   Main file path: app.py

5. Advanced settings 클릭

6. Secrets 설정
   다음 내용 복사해서 붙여넣기:
   
   ┌─────────────────────────────────────┐
   │ OPENROUTER_API_KEY = "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" │
   └─────────────────────────────────────┘

7. "Deploy!" 클릭

8. 배포 진행 확인
   - Installing dependencies... (1분)
   - Running app... (1분)
   - App is live! (완료)

9. 완료!
   URL이 생성됩니다:
   https://school-chatbot-xxxxx.streamlit.app/

═══════════════════════════════════════════════
  ✅ STEP 6: 테스트 (2분)
═══════════════════════════════════════════════

1. 생성된 URL 클릭

2. 챗봇 화면 확인

3. 테스트 질문:
   "학교를 소개해주세요"
   "학교 위치 알려주세요"

4. 정상 작동 확인!

═══════════════════════════════════════════════
  🎉 완료!
═══════════════════════════════════════════════

이제 다음을 하실 수 있습니다:

✅ 챗봇 URL 학교 홈페이지에 링크
✅ 학생/학부모에게 공유
✅ QR 코드 생성
✅ 입학 설명회에서 시연

═══════════════════════════════════════════════
  🆘 문제 발생 시
═══════════════════════════════════════════════

문제 1: "파일 업로드가 안 돼요"
해결: 
- 파일 크기 확인 (50MB 이하)
- 인터넷 연결 확인
- 브라우저 새로고침

문제 2: "API 키 오류가 나요"
해결:
- Streamlit Cloud → Settings → Secrets 확인
- API 키 정확히 입력했는지 확인

문제 3: "PDF를 못 읽어요"
해결:
- data/school_docs/ 폴더에 PDF 있는지 확인
- PDF 파일 열어서 내용 확인
